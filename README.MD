### Алгоритм решения

Программа начинает работу с создания объекта Human : с помощью методов класса Input запрашивает у пользователя вес, рост и возраст (метод InputUInt требует ввода натурального числа и повторяет запрос при ошибке). На основе введённых данных создаётся объект human_1, выводится его строковое описание и произведение веса, роста и возраста через метод Multiplication(). Затем создаётся объект human_2 как копия human_1 с помощью конструктора копирования, и для него также выводятся данные. Далее создаётся студент (Student): запрашиваются имя (InputString проверяет, что строка не пуста), курс и наличие долгов (InputBool требует ввода true или false). Объект student_1 создаётся через конструктор с параметрами (курс, долги, имя, вес, рост, возраст), после чего вызывается его метод NextCourse(), который проверяет: если нет долгов и курс меньше 4 — выводит сообщение о переводе на следующий курс, иначе — сообщение об отказе в переводе и вызывает метод DeleteStudent(), выводящий информацию об отчислении при наличии долгов. Затем создаётся второй студент student_2 с помощью конструктора, который копирует имя и курс из student_1, меняет значение долга на противоположное, а базовую часть (вес, рост, возраст) берёт из human_2. Для него также вызывается метод NextCourse(), который выполняет ту же логику перевода или отчисления.

# Задание 2

## Задача 1

### Текст задачи

Вычитание переменной типа Time
(учесть, что возможен переход в
предыдущие сутки). Результат должен
быть типа Time.

Унарные операции:
++ добавление минуты к объекту типа Time.
-- вычитание минуты из объекта типа Time
Операции приведения типа:
int (неявная) – результатом является количество минут (время
переводится в минуты);
bool (явная) – результатом является true, если часы и минуты не равны
нулю, и falseв противном случае.
Бинарные операции:
< Timet1, Timet2 — время переводится в минуты, результатом является
true, если количество минут в левом операнде меньше, чем количество
минут в правом операнде, и false–в противном случае.
> Timet1, Timet2 — время переводится в минуты, результатом является
true, если количество минут в левом операнде больше, чем количество
минут в правом операнде, и false–в противном случае. 

### Алгоритм решения

Вспомогательный класс Input содержит два статических метода для безопасного ввода значений. Метод InputHour выводит сообщение и в цикле запрашивает у пользователя число, проверяя с помощью byte.TryParse, что введено целое число в диапазоне от 0 до 24 включительно; при некорректном вводе выводится сообщение «Введите число от 0 до 24» и запрос повторяется. Метод InputMin аналогично запрашивает число от 0 до 60 включительно. В методе Main программы сначала создаются два объекта Time: для каждого с помощью методов InputHour и InputMin запрашиваются часы и минуты, полученные значения приводятся к типу byte и передаются в конструктор Time. Конструктор через свойства Hours и Minutes инициализирует поля _hours и _minutes, при этом сеттеры свойств проверяют, что переданное значение не превышает 23 для часов и 59 для минут. После создания двух объектов выполняется операция вычитания time1 - time2, для которой в классе Time перегружен бинарный оператор -: он преобразует часы и минут каждого времени в общее количество минут, вычисляет разность, и если результат отрицательный, прибавляет 1440 (количество минут в сутках), после чего преобразует полученное значение обратно в часы и минуты и возвращает новый объект Time. Результат вычитания вместе с исходными объектами выводится на экран с помощью метода ToString(), который возвращает строку в формате «ЧЧ:ММ» с ведущими нулями. Далее демонстрируется работа унарных операций: оператор ++ увеличивает время на одну минуту, при переполнении суток сбрасывает время на 00:00, оператор -- уменьшает на одну минуту, при отрицательном значении устанавливает время на 23:59. Результаты постфиксного инкремента и декремента выводятся на экран. Затем показываются операции приведения типов: неявное приведение Time к int возвращает общее количество минут от полуночи, явное приведение к bool возвращает false, если время равно 00:00, и true в противном случае. Наконец, демонстрируются перегруженные бинарные операторы сравнения > и <, которые сравнивают два времени по общему количеству минут и возвращают логическое значение. Все результаты операций последовательно выводятся в консоль.
