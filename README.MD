## Задача 1

### Текст задачи

Разработать класс с тремя целочисленными полями. Создать конструктор копирования.
Разработать метод, вычисляющий произведение полей. Перегрузить метод ToString() для
формирования строки из полей класса. Реализовать дочерний класс (его содержание предложить
самостоятельно и описать в решении: какой содержательный смысл имеют поля; реализовать
конструкторы; предложить и реализовать 2-3 метода). Протестировать все конструкторы и
другие методы базового и дочернего классов.


### Алгоритм решения

Класс Human содержит три приватных поля: _weight , _height  и _age . В классе реализованы три конструктора: конструктор по умолчанию, устанавливающий вес 10, рост 50 и возраст 1; конструктор с параметрами, принимающий вес, рост и возраст и инициализирующий соответствующие поля; конструктор копирования, создающий новый объект на основе существующего. Для доступа к полям определены свойства Weight, Height и Age, каждое из которых содержит геттер и сеттер. Метод Multiplication() вычисляет и возвращает произведение веса, роста и возраста.
Класс Student является наследником класса Human и добавляет три приватных поля: _name (имя студента), _course (номер курса) и _debt (наличие академических задолженностей). В классе реализованы три конструктора: конструктор по умолчанию, вызывающий базовый конструктор Human() и устанавливающий имя «Петр», курс 1 и отсутствие долгов; конструктор с параметрами, принимающий курс, наличие долгов, имя, вес, рост и возраст, передающий вес, рост и возраст в базовый класс через ключевое слово base; конструктор копирования, принимающий объект Student и объект Human, копирующий имя и курс из студента, устанавливающий долг противоположным исходному значению, а базовую часть инициализирующий от переданного объекта Human. Для доступа к полям определены свойства Name, Course и Debt с геттерами и сеттерами. Метод DeleteStudent() проверяет наличие долгов и выводит сообщение об отчислении или его отсутствии. Метод NextCourse() проверяет, нет ли долгов и курс меньше 4: если условие истинно, выводится сообщение о переводе на следующий курс; если нет — выводится сообщение об отказе в переводе и вызывается метод DeleteStudent().

# Задание 2

## Задача 1

### Текст задачи

Вычитание переменной типа Time
(учесть, что возможен переход в
предыдущие сутки). Результат должен
быть типа Time.

Унарные операции:
++ добавление минуты к объекту типа Time.
-- вычитание минуты из объекта типа Time
Операции приведения типа:
int (неявная) – результатом является количество минут (время
переводится в минуты);
bool (явная) – результатом является true, если часы и минуты не равны
нулю, и falseв противном случае.
Бинарные операции:
< Timet1, Timet2 — время переводится в минуты, результатом является
true, если количество минут в левом операнде меньше, чем количество
минут в правом операнде, и false–в противном случае.
> Timet1, Timet2 — время переводится в минуты, результатом является
true, если количество минут в левом операнде больше, чем количество
минут в правом операнде, и false–в противном случае. 

### Алгоритм решения

В классе Time реализован конструктор, принимающий часы и минуты и инициализирующий поля через свойства Hours и Minutes, которые в своих сеттерах проверяют, что часы не превышают 23, а минуты — 59. Перегружен бинарный оператор operator -, который переводит оба времени в минуты, вычисляет разность, при отрицательном результате прибавляет 1440 минуты и возвращает новый объект Time. Перегружены унарные операторы operator ++ и operator --, которые увеличивают или уменьшают время на одну минуту, корректно обрабатывая переполнение суток. Добавлены операторы приведения типов: неявный operator int возвращает общее количество минут от полуночи, а явный operator bool возвращает false только для времени 00:00. Перегружены бинарные операторы сравнения operator < и operator >, сравнивающие два времени по общему количеству минут. Также переопределён метод ToString, возвращающий время в формате ЧЧ:ММ с ведущими нулями.
